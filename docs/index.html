01Premise

Below the netfilter layer.

Aquifer is a post-exploitation framework, not initial access. It assumes a foothold already exists and focuses on what comes next — hiding deeper than userspace tooling can see, and surviving against EDR, DFIR, and active hunt teams on Linux hosts.

The implant operates inside its own kernel-enforced namespace sandbox with a veth pair to host network and DNAT loopback for C2. Configuration bakes at compile time via -ldflags — no runtime args, no environment lookups, no secrets on disk.

03Quickstart

Build, verify, deploy.

Three build profiles: dev, garble (obfuscated), release (garble + UPX + scrubbing). Two-stage execution — parent validates guardrails and re-execs into namespace child.

# Prerequisites: Go 1.21+ (1.25+ for garble), Linux
$ git clone https://github.com/Real-Fruit-Snacks/Aquifer && cd Aquifer
$ make build              # dev: stock Go, debug symbols

# Hardened build profiles
$ make build-garble       # obfuscated: garble strips identifiers
$ make build-release      # garble + UPX + signature patching

# Verification
$ make opsec-check        # No Go metadata leaks · high entropy · UPX scrubbed
$ make strings-check      # Compare regular vs garble side-by-side

05Architecture

Two-stage exec, kernel-level hide.

Parent process validates guardrails and sets up host-side veth pair. Re-execs into the namespace child which operates inside isolation. Compile-time config via -ldflags. Memory-first design with ProtectedConfig and deterministic shredding.

06Authorization

Without permission, don't.

Aquifer is built for engagements that are scoped, written, and signed. Authorization required — designed exclusively for authorized security testing with explicit written permission. Unauthorized access to computer systems is illegal.

Vulnerabilities go through private security advisories, never public issues. 90-day responsible disclosure.

Aquifer does not scan or exploit (not initial access), spray credentials (not lateral movement), or operate cross-platform (Linux namespace isolation by design).
